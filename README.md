### üìù Description
This project was specifically designed to practice **server-side logic and data persistence using the File System (JSON files)** instead of a traditional database. It allows users to register, log in, and manage blog posts, with all data being handled through structured JSON storage.

> **Note:** This project was designed and optimized for **Desktop view** (monitors/laptops) and is not fully mobile responsive yet.

### ‚ú® Key Features
* **JSON Data Persistence:** Deep dive into Node.js `fs` module to read, write, and update users and posts stored in JSON files.
* **User Authentication:** Registration and login system with session-based logic.
* **Content Management:** Full ability to create and display blog posts on the main feed.
* **Dynamic Routing:** Individual views for each post and dedicated "About", "Contact", "Login" and "Register" pages.

### üöÄ Getting Started

To run this project locally, follow these steps:

1. **Clone the repository:**
   ```bash
   git clone [git@github.com:PrzemekO84/CapStoneProjectBlog.git](https://github.com/PrzemekO84/CapStoneProjectBlog/tree/main)
2. **Install dependencies:**
   ```bash
   npm install
3. **Start the server: Using nodemon (for development):**
   ```bash
   nodemon index.js
4. Or using standard node:
   ```bash
   node index.js
5. **Open in browser: Go to http://localhost:3000**
